# 전투 시스템 가이드

이 문서는 브라운더스트 전투 시뮬레이터의 전투 메커니즘을 상세하게 설명합니다.

---

## 1. 턴 진행 순서

한 용병의 턴은 다음 순서로 진행됩니다:

```
[1] 턴 시작 처리
  → 지속 피해(DoT) 적용 (화상/출혈/중독)
  → 재생(regen) 회복
  → 상태 효과 턴수 감소, 만료된 효과 제거
  → 기절/빙결/감전이면 턴 스킵 → 종료

[2] 일반공격 전 스킬
  → 침묵/현혹이면 스킬 발동 안됨
  → 이로운 효과(버프) 먼저 → 해로운 효과(디버프) 나중에
  → 같은 분류 내에서는 스킬 번호 순서대로

[3] 일반공격
  → 버프/디버프가 반영된 스탯으로 계산
  → 치명타/스침 판정
  → 보호막 감소 적용
  → 흡혈 처리
  → 반사반격 처리

[4] 일반공격 후 스킬
  → 침묵/현혹이면 스킬 발동 안됨
  → 스킬 번호 순서대로 발동
```

### passive 스킬

passive 스킬은 **전투 시작 전에 1회 발동**됩니다. 모든 용병의 passive 스킬이 일괄 적용된 후 전투가 시작됩니다.

---

## 2. 데미지 계산

### 기본 공식

```
총 데미지 = 가변 데미지 + 고정 데미지

가변 데미지 = ATK × (1 - fixedDamageRate) × (1 - DEF/100) × 스킬배율 × 치명배율 × 스침배율 × 받피증 × 보호막
고정 데미지 = ATK × fixedDamageRate × 스킬배율 × 치명배율 × 받피증 × 보호막
```

- 고정 데미지는 **DEF와 스침(graze)의 영향을 받지 않음**
- `fixedDamageRate` 미지정 시 0 (전부 가변 데미지)
- 모든 계산은 **% 기반**입니다 (절대값 차감이 아님)
- DEF가 100% 이상이면 가변 데미지는 **0**입니다
- 결과는 반올림 처리됩니다

### 유효 ATK (버프/디버프 반영) — 패턴 A

```
multiply = clamp(승산버프합 - 승산디버프합, -0.8, 3.0)
유효 ATK = max(0, 기본 ATK × (1 + multiply / 100)) + plus
```

- 버프/디버프의 `channel` 필드로 승산(multiply)/가산(plus) 구분 (미지정 시 multiply)
- 승산 클램프: **[-80%, +300%]** (기본 ATK의 -80%~+300%)
- 가산(plus)은 승산 적용 후 별도 가산

### 유효 DEF (버프/디버프 반영)

```
유효 DEF = max(0, 기본 DEF% + 방어력 증가% - 방어력 감소%)
```

- DEF 자체가 %이며, 버프/디버프는 %p 단위로 가감
- 예: DEF 25% + def_up 10 → 35%

### 치명타

치명타 확률에 따라 발동합니다. **WELL512 RNG(만분율)** 기반으로 판정됩니다.

```
크리티컬 배율 = max(0, (2.0 + 치명피해/100) × (1 - critResist))
```

- `critResist` (0.0~1.0): 대상의 크리티컬 저항. 크리티컬 배율을 감소시킴
- 예: 치명피해 20%, critResist 0 → 치명타 시 2.2배 데미지
- 예: 치명피해 20%, critResist 0.3 → 치명타 시 2.2 × 0.7 = 1.54배 데미지

치명확률: 패턴 B 공식 (비율 스탯)
```
유효 치명확률 = clamp((기본치명확률 + plus) × (1 + multiply), 0, 1)
```

치명피해: 패턴 B 공식 (상한 없음)
```
유효 치명피해 = max(0, (기본치명피해 + plus) × (1 + multiply))
```

### 스침 (Graze)

피격자의 **민첩** 수치에 따라 발동합니다. **WELL512 RNG(만분율)** 기반으로 판정됩니다.

```
스침 발동 확률 = 유효 민첩 (%)
스침 시 가변 데미지 × 0.65 (35% 감소), 고정 데미지는 영향 없음
```

- 치명타와 스침은 **동시에 발동 가능**
- 민첩: 패턴 B 공식 (비율 스탯)
  ```
  유효 민첩 = clamp((기본민첩 + plus) × (1 + multiply), 0, 1)
  ```

### 보호막 감소

방어자에게 보호막(shield) 버프가 있으면 최종 데미지가 추가 감소됩니다.

```
보호막 감소율 = (1 - 보호막1값/100) × (1 - 보호막2값/100) × ...
```

- **곱연산**으로 적용
- **상한: 최대 70% 감소** (감소율 최소 0.3)
- 예: 100% 보호막 2개 → 감소율 = (1-1) × (1-1) = 0 → 상한 적용 → 0.3 (70% 감소)

### DEF 스케일링 데미지 (철갑)

대상의 DEF를 무시하고, 공격자의 DEF와 ATK를 기반으로 추가피해를 입힙니다.

```
추가피해 = (공격자 유효DEF / 100) × 공격자 유효ATK × (스킬 value / 100)
```

- 치명타/스침 판정 적용
- 대상 DEF 무시 (대상의 보호막/받피증은 적용)
- 예: DEF 12, ATK 874, value 275 → (12/100) × 874 × (275/100) ≈ 288 추가피해

### 받는 피해량 증가

방어자에게 `dmg_taken_up` 디버프가 있으면 최종 데미지가 증가합니다.

```
받는 피해량 배율 = 1 + (받는 피해량 증가 합계 / 100)
```

## 3. 스탯 상세

### 기본 스탯

| 스탯 | 설명 | 비고 |
|------|------|------|
| **HP** | 체력. 0이 되면 전투 불능 | - |
| **ATK** | 공격력. 데미지의 기본값 | 버프는 기본 ATK의 %, 상한 +300% |
| **DEF** | 방어력 (%). 받는 데미지를 % 감소 | 100%면 데미지 0 |
| **치명확률** | 치명타 발동 확률 (%) | 버프/디버프로 증감 |
| **치명피해** | 치명타 시 추가 배율 (%) | 기본 2.0 + 이 수치/100 |
| **민첩** | 피격 시 특수 효과 발동 기준 | 아래 참조 |

### 민첩의 효과

민첩은 **피격 시** 두 가지 효과를 제공합니다:

1. **스침 확률**: 민첩 수치 = 스침 발동 확률 (%). 스침 시 데미지 35% 감소
2. **디버프 턴수 감소**: 디버프를 받을 때 지속 턴수가 **50% 감소** (최소 1턴)

예: 민첩 20인 수호자
- 20% 확률로 스침 발동 (피격 데미지 35% 감소)
- 3턴 디버프 → 2턴으로 감소 (3 × 0.5 = 1.5 → 반올림 2)

## 4. 스킬 시스템

### 스킬 타이밍

각 스킬은 다음 중 하나의 발동 시점을 가집니다:

| 타이밍 | 발동 시점 | 설명 |
|--------|----------|------|
| `before_attack` | 일반공격 전 | 이로운 효과 먼저, 해로운 효과 나중에 발동 |
| `after_attack` | 일반공격 후 | 스킬 번호 순서대로 발동 |
| `passive` | 전투 시작 전 | 1회 발동 후 종료 |

### 일반공격 전 스킬의 순서 규칙

일반공격 전(before_attack) 스킬은 **이로운 효과 → 해로운 효과** 순서로 발동됩니다.
스킬의 원래 번호 순서는 무시되고, 이로운/해로운 분류가 우선합니다.

예: 스킬이 [디버프, 버프, 디버프, 버프] 순서라면
→ 실제 발동: [버프, 버프, 디버프, 디버프]

### 스킬 대상

스킬의 대상은 용병의 `attackTarget`에 의해 결정됩니다. 적 대상 스킬은 용병의 타겟/범위를 따릅니다.

| 대상 | 설명 |
|------|------|
| `enemy_front` | 적 진영에서 가장 앞에 있는 적 |
| `enemy_second` | 맨앞 적을 건너뛰고 2번째 적을 공격 |
| `enemy_back` | 적 진영에서 가장 뒤에 있는 적 |
| `enemy_random` | 살아있는 적 중 랜덤 |
| `next_ally` | 자신의 다음 행동 순서인 아군 |
| `self` | 자기 자신 |

모든 적 타겟 유형은 동일한 행 탐색 순서를 따릅니다: **같은 행 → 아래행 → 윗행**

### 공격 범위

범위와 타겟은 **스킬이 아닌 용병**에 속합니다. 동일 스킬이라도 용병의 범위를 따르며, 일반공격뿐 아니라 모든 스킬이 해당 용병의 타겟/범위를 사용합니다.

| 범위 | size | 설명 |
|------|------|------|
| `single` | X | 단일 타겟 |
| `horizontal` | O | 앞뒤 n칸 |
| `vertical` | O | 위아래 n칸 |
| `back_n` | O | 뒤로 n칸 |
| `front_n` | O | 앞으로 n칸 |
| `cross` | O | 십자형 (위아래 size칸 + 앞뒤 1칸) |
| `x_shape` | X | 대각선 X자 |
| `area_n` | O | n*n 정사각형 |
| `diamond` | X | 마름모꼴 (맨해튼 거리 2) |
| `small_cross` | X | + 형태 (자기 칸 + 상하좌우 1칸) |

### 다중 타겟 처리

범위 공격 시 대상을 **순차 처리**합니다 (동시가 아님):

- 메인 타겟 → 범위 내 다음 대상 → 반복
- **치명/스침은 대상별 개별 판정**
- 쓰러진 대상은 스킵
- 1번 대상 공격 시 디버프 반격이 발동하면, 2번 이후 공격에도 해당 디버프가 누적 적용
- 반사반격으로 공격자가 쓰러지면 나머지 대상 공격 중단

### 즉시 효과 vs 지속 효과

- **즉시 효과**: `duration` 없음. 바로 적용 (damage, heal, heal_percent, dispel, cleanse 등)
- **지속 효과**: `duration`이 있으면 상태 효과(버프/디버프)로 대상에게 부여됨
- **회복(heal/heal_percent)도 치명/스침 판정**을 받음. 치명 시 회복량 증가, 스침 시 감소

### 스킬 체이닝 (triggerSkill)

스킬 효과에 `triggerSkill` 필드를 지정하면, 해당 효과 적용 후 다른 스킬을 연쇄 발동합니다.

- 부가 스킬은 상위 스킬과 **동일한 대상**에게 발동
- 향후 다양한 "스킬 발동 시 추가 스킬" 패턴에 재사용 가능

예: 상급 무효화 → 버프 제거 + `triggerSkill: 'buff_block'` → 이로운 효과 금지 연쇄 발동

### 지원력 스케일링 (spScaling)

스킬 효과에 `spScaling: true`가 지정되면, `value`를 시전자의 `supportPower` 기준으로 스케일링합니다.

```
실제 value = Math.round(시전자 supportPower × value / 100)
```

- 지원형 용병은 ATK=0이므로, `atkScaling` 대신 `spScaling`을 사용
- 예: supportPower=203, value=85 → 203 × 85 / 100 = 172.55 → 173 공격력 증가

### 연동 버프 제거 (linkedBuffId)

스킬 효과에 `linkedBuffId`가 지정되면, 해당 상태효과가 제거될 때 `linkedBuffId`에 지정된 type의 버프도 함께 제거됩니다.

- 같은 스킬에서 동시에 부여된 버프 간의 연동에 사용
- 예: 정정당당의 `temp_hp` 효과에 `linkedBuffId: 'atk_up'` → 임시생명력이 소진/만료되면 공격력 증가도 함께 제거

## 5. 버프 시스템

### 버프 3분류

버프는 제거 규칙에 따라 3가지로 나뉩니다:

#### (1) 보호막 (shield)

받는 피해량을 감소시키는 효과입니다.

- 여러 보호막 적용 시 **곱연산**
- **상한**: 최대 70% 감소 (어떤 조합이든 70%를 넘길 수 없음)

#### (2) 능력치 강화 (stat_enhance)

스탯을 직접 증가시키는 버프입니다.

- 여러 개 적용 시 **합연산**
- **스탯별 클램프 규칙**:

| 스탯 | 승산(multiply) 클램프 | 결과 클램프 |
|------|------|------|
| 공격력 (ATK) | [-0.8, 3.0] | >= 0 |
| 치명확률 | 없음 | [0, 1] |
| 치명피해 | 없음 | >= 0 (상한 없음) |
| 민첩 | 없음 | [0, 1] |
| 방어력 | 없음 | >= 0 |

#### (3) 특별 버프 (special)

특수한 제거 규칙을 가진 버프입니다.

- **강화무효화로 제거 불가** — 오직 **무효화(dispel)**로만 제거 가능

### 버프 효과 종류

| 효과 | 분류 | 설명 |
|------|------|------|
| `atk_up` | stat_enhance | 공격력 증가 |
| `def_up` | stat_enhance | 방어력 증가 |
| `crit_up` | stat_enhance | 치명확률 증가 |
| `temp_hp` | stat_enhance | 임시생명력 (아래 참조) |
| `regen` | - | 매 턴 HP 회복 |
| `lifesteal` | - | 일반공격 데미지의 일정% 회복 |
| `counter_damage` | - | 피격 시 받은 데미지의 일정% 반격 |
| `counter_debuff` | - | 피격 시 공격자의 강화 버프 제거 |
| `reflect_debuff` | - | 받는 디버프를 공격자에게 반사 |
| `debuff_immune` 계열 | - | 디버프 면역 (아래 면역 시스템 참조) |

### 임시생명력 (temp_hp)

현재 HP 위에 추가 HP를 부여하는 버프입니다. `BattleCharacter.tempHp`에 저장됩니다.

```
임시생명력 = 시전자 maxHp × (시전자 supportPower / 100) × (value / 100)
```

- 피격 시 임시생명력이 **hp보다 먼저 소진**됨
- 임시생명력이 0이 되거나 지속턴이 만료되면 제거
- `linkedBuffId`가 지정된 경우, 제거 시 연동된 버프도 함께 제거됨
- 예: 아리네스(supportPower=203, maxHp=3395) → 대상에게 3395 × 2.03 × 1.0 = 6892 임시생명력 부여

## 6. 디버프 시스템

### 디버프 3분류

디버프는 면역 시스템과 연동되며, 3가지로 분류됩니다.

#### (1) 공격방해 (CC)

정상적인 행동을 방해하는 효과입니다.

| 효과 | 이름 | 결과 |
|------|------|------|
| `stun` | 기절 | 턴 전체 스킵 (행동 불가) |
| `freeze` | 빙결 | 턴 전체 스킵 (행동 불가) |
| `shock` | 감전 | 턴 전체 스킵 (행동 불가) |
| `silence` | 침묵 | 스킬 발동 불가 (일반공격만 가능) |
| `charm` | 현혹 | 아군을 공격하게 됨 |

CC 효과의 행동 제어:
- **기절/빙결/감전**: 턴 시작 직후 행동 불가 → 턴 종료
- **침묵**: 스킬 발동 불가, 일반공격은 가능
- **현혹**: 적 대신 아군을 공격함 (스킬 발동 불가)

#### (2) 지속 피해 (DoT)

매 턴 시작 시 피해를 입히는 효과입니다.

| 효과 | 이름 | 결과 |
|------|------|------|
| `burn` | 화상 | 매 턴 고정 피해 |
| `advanced_burn` | 상급 화상 | 매 턴 고정 피해 (ATK 스케일링 + 받피증) |
| `bleed` | 출혈 | 매 턴 고정 피해 |
| `poison` | 중독 | 매 턴 고정 피해 |

DoT 피해는 **턴 시작 시** 처리되며, 방어력/보호막의 영향을 받지 않습니다.

#### (3) 능력치 약화 (stat_weaken)

스탯을 감소시키는 효과입니다.

| 효과 | 이름 | 결과 |
|------|------|------|
| `atk_down` | 공격력 감소 | ATK 하락 |
| `def_down` | 방어력 감소 | DEF 하락 |
| `crit_rate_down` | 치명확률 감소 | 치명확률 하락 |
| `crit_damage_down` | 치명피해 감소 | 치명타 추가 배율 하락 |
| `agility_down` | 민첩 감소 | 스침 확률 및 디버프 턴수 감소 효과 하락 |
| `dmg_taken_up` | 받는 피해량 증가 | 최종 데미지 증가 |
| `curse` | 저주 | 회복량 감소 |
| `buff_block` | 이로운 효과 금지 | 새로운 버프를 받을 수 없음 |

## 7. 면역 시스템

면역은 새로운 디버프(또는 버프)의 **적용 자체를 차단**합니다.

### 면역 5분류

낮은 단계부터 높은 단계 순서로:

#### (1) 공격방해 면역 (`cc_immune`)
→ CC 분류 디버프만 차단 (기절, 빙결, 감전, 침묵, 현혹)

#### (2) 지속피해 면역 (`dot_immune`)
→ DoT 분류 디버프만 차단 (화상, 출혈, 중독)

#### (3) 능력치약화 면역 (`stat_debuff_immune`)
→ stat_weaken 분류 디버프 전체 차단 (공격력/방어력/치명확률/치명피해/민첩 감소, 받는 피해량 증가, 저주, 일점사, 이로운 효과 금지)

#### (4) 해로운효과 면역 (`harmful_immune`)
→ 위 3가지 분류에 해당하는 **모든** 디버프를 차단
→ 단, 스킬에 `ignoreImmunity` 속성이 있으면 **관통**됨

#### (5) 모든효과 면역 (`all_immune`)
→ 디버프뿐 아니라 **아군의 버프도 받을 수 없음**
→ 완전한 효과 차단 상태

### 면역 판정 흐름

```
효과 적용 시도
  → 대상에 all_immune 있음? → 버프/디버프 모두 차단
  → 버프인가? → all_immune 외에는 차단 안됨 → 정상 적용
  → 디버프인데 ignoreImmunity인가? → 면역 관통 → 정상 적용
  → 대상에 harmful_immune 있음? → 차단
  → 대상에 세부 면역(cc/dot/stat_debuff) 있음? → 해당 분류만 차단
  → 해당 없음 → 정상 적용
```

### 면역과 무효화는 별개

- **면역**: 새로운 효과의 **적용**을 차단
- **무효화(dispel)**: 이미 걸린 버프를 **제거**
- 면역 상태여도 무효화는 통함 (서로 간섭하지 않음)

---

## 8. 무효화 & 정화

### 무효화 (Dispel)

적에게 걸린 **버프를 제거**하는 효과입니다. 두 가지 등급이 있습니다:

| 종류 | 보호막 | 능력치 강화 | 특별 버프 | 면역 버프 |
|------|--------|------------|----------|----------|
| **강화무효화** (`dispel_enhance`) | 제거 | 제거 | **제거 불가** | 제거 불가 |
| **무효화** (`dispel`) | 제거 | 제거 | **제거** | 제거 불가 |

- 제거 갯수 제한 없음 (조건에 맞는 버프 모두 제거)
- 면역 버프(`_immune` 계열)는 어떤 무효화로도 제거 불가

### 정화 (Cleanse)

아군에게 걸린 **디버프를 제거**하는 효과입니다.

- 제거 갯수 제한 없음 (걸린 디버프 모두 제거)

## 9. 반사반격 시스템

일반공격으로 피격당했을 때 자동으로 발동하는 효과입니다.

### 반사반격 3유형

#### (1) 데미지 반격 (`counter_damage`)

피격 시 공격자에게 **받은 데미지의 일정%**를 되돌립니다.

```
반격 데미지 = 받은 데미지 × (counter_damage값 / 100)
```

#### (2) 디버프 반격 (`counter_debuff`)

피격 시 공격자의 **강화 버프를 제거**합니다 (강화무효화와 동일한 효과).
→ 특별 버프는 제거하지 못함

#### (3) 디버프 반사 (`reflect_debuff`)

받는 디버프를 공격자에게 되돌리는 효과입니다.
→ **현재 미구현**: 로그만 출력되며 실제 디버프 반사는 동작하지 않음

### 반사반격 특이사항

- 현혹 상태에서 아군을 공격할 때는 반사반격이 발동하지 않음
- 방어자가 이미 쓰러진 경우 반사반격이 발동하지 않음

## 10. 타겟 강제 변경

용병의 `attackTarget`에 의한 기본 타겟 선택을 강제로 변경하는 메커니즘입니다.

### 도발 (Taunt)

방어형 용병이 적의 공격을 자신에게 끌어오는 **버프**입니다.

- 대상: 자기 자신에게 부여
- 효과: 도발이 걸린 용병이 살아있으면, 적이 이 용병을 최우선으로 공격
- 아군을 보호하기 위한 방어적 메커니즘

### 일점사 (Focus Fire)

특정 적에게 거는 **디버프**로, 아군 전체가 해당 적을 집중 공격하게 만듭니다.

- 대상: 적에게 부여
- 효과: 일점사가 걸린 적이 살아있으면, 아군이 그 적을 최우선으로 공격
- `resolveEnemyTarget()`에서 적 중 `focus_fire` 상태인 대상이 있으면 메인 타겟으로 반환
- 공격적 디버프 — 원하는 적을 집중 제거하기 위한 메커니즘

### 도발 무시 (Taunt Immune)

도발을 무시하는 **버프**입니다.

- 대상: 자기 자신에게 부여 (보통 passive)
- 효과: 적의 도발을 무시하고 자신의 `attackTarget`대로 타겟 선택
- **일점사는 무시하지 않음** — 일점사 대상이 있으면 그쪽으로 공격

### 우선순위

타겟 선택 시 강제 변경의 우선순위:

```
1. 일점사 (focus_fire) — 최우선. 도발 무시로도 무시 불가
2. 도발 (taunt) — 도발 무시 버프가 있으면 무시됨
3. 기본 타겟 방식 (attackTarget) — 위 둘 다 없을 때
```

## 11. 상태 효과 턴 관리

### 턴수 감소 시점

상태 효과의 남은 턴수는 **해당 캐릭터의 턴 시작 시** 1씩 감소합니다.
턴수가 0이 되면 즉시 제거됩니다.

### 민첩에 의한 턴수 감소

민첩이 있는 캐릭터가 디버프를 받으면, 적용되는 턴수가 **50% 감소**됩니다 (최소 1턴).

```
실제 턴수 = max(1, round(원래 턴수 × 0.5))
```

예: 민첩 20인 수호자에게 기절 3턴 → 실제 적용 2턴

### DoT와 재생의 처리 순서

턴 시작 시 처리 순서:
1. 모든 DoT 피해 적용 (화상/출혈/중독)
2. 재생(regen) 회복 적용
3. 턴수 감소 및 만료 효과 제거
4. CC 판정 (기절/빙결/감전이면 턴 스킵)

## 12. RNG 시스템 (결정론적 전투)

### WELL512 PRNG

전투의 모든 무작위 판정은 **WELL512a 알고리즘** 기반 RNG를 사용합니다. 동일한 시드를 사용하면 항상 동일한 전투 결과가 보장됩니다.

```typescript
// 시드 지정 시 결정론적 전투
simulateBattle(team1, team2, seed)

// 시드 미지정 시 랜덤 생성
simulateBattle(team1, team2)
```

사용된 시드는 전투 로그의 첫 번째 `round_start` 항목에 기록됩니다.

### 만분율 기반 판정

원본 게임과 동일하게 **만분율(0~9999)** 기반으로 판정합니다:

```
치명타 판정: rng(0~9999) < 치명확률 × 100
스침 판정:   rng(0~9999) < 민첩 × 100

예: 치명확률 7.5% → 만분율 750 → rng < 750이면 치명타
```

### RNG 미결정 사항

원본 게임 내부 WELL512 상태 배열이 64비트(`long[]`)일 가능성이 있음 (RVA `0x1FC47F8` 미디컴파일). 현재 구현은 표준 32비트를 채택했으며, 원본과의 시퀀스 완전 일치는 미검증 상태.

## 13. 화염 가드 (Flame Guard)

### 개요

`flame_guard` 상태효과가 있는 방어자를 공격하면, 데미지 일부가 흡수되어 공격자에게 반사됩니다.

```
흡수량 = 피격 데미지 × (flame_guard.value / 100)
실제 피격 데미지 = 원래 데미지 - 흡수량
반사 데미지 = 흡수량 (공격자에게 직접 적용)
```

- 일반공격, 스킬 데미지 모두 적용
- 반사 데미지로 공격자가 사망하면 사망 판정 체인 발동
- DoT 등 간접 피해에는 적용되지 않음

### 상태 효과 예시

```typescript
{
  type: 'flame_guard',
  value: 30,   // 30% 흡수 + 반사
  duration: 6,
  category: 'buff',
  buffType: 'special',
}
```

## 14. 피격 콜백 (Wounded Add Buff)

### 개요

`wounded_add_buff` 상태효과가 있는 캐릭터가 직접 공격을 받으면, 연동된 스킬을 자신에게 자동 발동합니다.

```
피격 시 → wounded_add_buff 확인 → linkedBuffId 스킬 조회 → 자신에게 효과 적용
```

- 일반공격, 스킬 데미지로 피격 시 발동 (DoT 제외)
- 공격자가 아닌 **자기 자신**에게 스킬 효과 적용
- 여러 개 중첩 시 모두 발동

### 상태 효과 예시

```typescript
{
  type: 'wounded_add_buff',
  value: 0,
  duration: 99,
  category: 'buff',
  buffType: 'special',
  linkedBuffId: 'some_skill_id',  // 피격 시 발동할 스킬 ID
}
```
