# 배포 가이드

## 배포 플랫폼: Vercel

이 프로젝트는 백엔드 없는 순수 클라이언트 앱이므로, 정적 호스팅으로 배포합니다.

## 최초 배포

### 1. GitHub 레포 Public 전환

1. GitHub 레포 페이지 → **Settings** 탭
2. 맨 아래 **Danger Zone** → **Change repository visibility**
3. **Make public** 선택 → 레포 이름 입력하여 확인

### 2. Vercel 연동

1. [vercel.com](https://vercel.com) 접속 → **Sign Up / Log In with GitHub**
2. **Add New Project** → 레포 목록에서 `app-brown-dust` 선택
3. Framework Preset이 **Vite**로 자동 감지되는지 확인
4. **Deploy** 클릭

완료되면 `https://app-brown-dust.vercel.app` 같은 URL이 발급됩니다.

## 업데이트 배포

Vercel 연동 후에는 다음만 하면 자동 배포됩니다:

```bash
git add <변경된 파일>
git commit -m "커밋 메시지"
git push
```

push를 감지하여 Vercel이 자동으로 재빌드 + 재배포합니다.

## 배포 전 확인사항

```bash
# 타입 체크 + 빌드가 정상인지 확인
npm run build
```

빌드가 실패하면 Vercel 배포도 실패합니다. push 전에 로컬에서 확인하세요.
