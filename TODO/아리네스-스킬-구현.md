# 아리네스 스킬 4종 구현 계획

## Context

아리네스(지원형, 3성)에 스킬이 없는 상태. 스크린샷 기반으로 4개 스킬을 구현한다.
지원형 전용 메카닉으로 **지원력 스케일링**, **임시생명력**, **조건부 버프 연동**이 필요.

## 스킬 분석 (스크린샷 기반)

| # | 이름 | 타이밍 | 대상 | 지속 | 핵심 효과 |
|---|------|--------|------|------|-----------|
| 1 | 정정당당 | after_attack | next_ally | 16턴 | 임시생명력(최대HP×지원력×100%) + 공격력 증가(지원력×85%) — 임시생명력 제거 시 공격력 증가도 종료 |
| 2 | 상급 공격력 증가 부여 | after_attack | next_ally | 12턴 | 공격력 증가(지원력×85%) + 치명확률 증가(지원력×20%) |
| 3 | 공격 방해 면역 | before_attack | self | 8턴 | 자신: cc_immune |
| 4 | 능력치 약화 면역 부여 | before_attack | next_ally | 8턴 | 아군: stat_debuff_immune |

> 3번과 4번은 대상이 다르므로 별도 스킬 2개로 등록 (방안 A 확정).

## 새로운 메카닉

### 1. 지원력 스케일링 (`spScaling`) — SkillEffect 확장

현재 `atkScaling: true`면 시전자 ATK 기준으로 value를 계산하는 패턴이 있음.
지원형은 ATK=0이므로, `supportPower` 기준 스케일링이 필요.

- `spScaling: true` → `value`를 `시전자 supportPower × value / 100`으로 계산
- 아리네스 기준: supportPower=203, value=85 → 203 × 85 / 100 = 172.55 → 173 공격력 증가
- 수정: `src/types/skill.ts` — SkillEffect에 `spScaling?: boolean` 추가
- 수정: `src/logic/turn.ts` — `applyStatusEffect()`에서 spScaling 처리

### 2. 임시생명력 (`temp_hp`) — 새 버프 타입

- 최대HP 위에 추가 HP를 부여 (현재 HP 상한을 초과 가능)
- 공식: `시전자 maxHp × 시전자 supportPower / 100 × value / 100`
- 아리네스 기준: 시전자 maxHp × 203/100 × 100/100 = 시전자 maxHp × 2.03
- 피격 시 임시생명력이 먼저 소진됨
- 임시생명력이 0이 되거나 지속턴 종료 시 연동된 atk_up도 함께 제거
- `buffType: 'stat_enhance'`
- 수정: `src/types/game.ts` — BattleCharacter에 `tempHp?: number` 추가
- 수정: `src/logic/turn.ts` — 데미지 적용 시 tempHp 우선 소진
- 수정: `src/logic/turn.ts` — 상태효과 만료/제거 시 연동 버프 함께 제거

### 3. 연동 버프 제거 (`linkedBuffId`) — SkillEffect 확장

정정당당의 핵심: 임시생명력이 사라지면 공격력 증가도 같이 사라짐.
- `linkedBuffId: string` — 이 효과가 제거될 때 함께 제거할 상태효과의 type
- temp_hp 효과에 `linkedBuffId: 'atk_up'` 지정
- 수정: `src/types/skill.ts` — SkillEffect에 `linkedBuffId?: string` 추가
- 수정: `src/logic/turn.ts` — 상태효과 제거 시 linkedBuffId 체크

  { type: 'atk_up', value: 85, duration: 16, buffType: 'stat_enhance', spScaling: true },

## 검증

1. `npx tsc --noEmit` — 타입 체크 통과
2. `npm run dev` — 아리네스 배치 후 전투 테스트
3. 공격력 증가: 아군 ATK가 지원력×85% 만큼 증가하는지 확인
4. 치명확률 증가: 아군 치명확률이 지원력×20% 만큼 증가하는지 확인
5. 임시생명력: 대상 HP가 maxHp를 초과하는지 확인
6. 연동 제거: 임시생명력 소진 시 공격력 증가도 제거되는지 확인
7. 면역: 자신 cc_immune, 아군 stat_debuff_immune 동작 확인
