## ⚠️ 설계 결정 필요 (작업 재개 시 먼저 결정)

### 원본 게임 스킬 구조 (2026-03-19 발견)

캐릭터당 **스킬 1개** (`CharBasicList._skill`), 그 안에 **효과 최대 4개** (`SkillBasicList._addBuff1~4`):

```
CharBasicList._skill = 67510  (코델리아)
  └─ SkillBasicList[67510]
       ├─ _type: 1 (attack) | _actionType: 2 (horizontal) | _searchType: 1 (enemy_front)
       ├─ _addBuff1: 6741001  → DotDamage/1 = 화상 25%, 5턴
       ├─ _addBuff2: 6740903  → DiedBuff/1 = 사망 콜백
       └─ _addBuff4: 6750007  → BuffProtect = 보호막 25%, 12턴
```

이 구조와 달리 **웹앱 현재 구조는 용병당 3~4개 Skill 객체** (수동 작성된 추상화).

### A안 — 현재 구조 유지 (용병당 3-4개 Skill 객체)

- 기존 10개 용병 스킬 수정 불필요
- 스킬 1개를 여러 Skill로 분리하는 변환 로직 필요
- 자동화 복잡도 높음

### B안 — 게임 구조로 단순화 (용병당 스킬 1개 + effects 배열)

- 기존 10개 용병 스킬 구조 전면 수정 필요
- 자동화 단순, 게임과 1:1 대응
- SkillTiming: `_type` → 1=attack, 2=support, 3=passive

**→ 결정 후 아래 작업 진행**
